üì° How It Works (Step by Step):

1. Setup Phase (via WiFi to your domain):

  • Phone 1 connects to your domain server via WiFi
  • Phone 1 joins a room (e.g., "ABC123")
  • Phone 2 connects to the same domain server via WiFi
  • Phone 2 joins the same room
  • Server matches them together

2. Connection Setup (signaling):

  • Phones exchange connection info through your server
  • This includes network addresses and connection details
  • Server acts as a "matchmaker" - just helps them find each other

3. Direct Connection (peer-to-peer):

  • Once connected, phones talk DIRECTLY to each other
  • Voice audio flows: Phone 1 ‚Üî Phone 2 (bypasses server!)
  • This is called WebRTC - Web Real-Time Communication
  • Low latency, efficient, secure

üí° Key Point: The server only helps establish the connection. Once connected, audio flows directly between phones via WiFi/mobile data - the server is no longer in the audio path!

üåê WiFi to Domain Setup:

  1. Deploy signaling server to your domain (see README)
  2. Enter domain URL: wss://yourdomain.com
  3. Both phones connect via WiFi to your domain
  4. Voice flows through WiFi, then directly between phones

üîµ Bluetooth Limitation:

Web browsers cannot directly stream audio via Bluetooth between phones. However, you can use Bluetooth tethering - one phone shares internet via Bluetooth, and WebRTC will use that connection.
